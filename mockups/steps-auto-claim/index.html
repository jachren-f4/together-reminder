Steps Auto-Claim

Celebrating yesterday's achievement when the app opens

The Problem

Users might forget to visit the "Steps Together" card and miss claiming their LP reward. The claim button is buried inside a feature screen.

The Solution: Auto-Claim on Display

LP is claimed automatically when the celebration overlay appears. No user action required to secure the reward.

  • First user: Opens app → overlay appears → LP auto-claimed → tap "Got it!" to dismiss
  • Second user: Opens app → sync shows partner claimed → celebration overlay → tap "Awesome!" to dismiss

Why Auto-Claim?

Eliminates risk of users dismissing overlay without claiming. Even if user force-closes the app, the LP is already added. The button purely dismisses the celebration - no action required to get the reward.

Technical Implementation

Preventing Duplicate Claims

  • Single source of truth: Server tracks `claimed` boolean and `claimedByUserId` per day
  • Auto-claim trigger: API call on overlay display, before showing UI. LP added atomically.
  • Steps screen sync: Same `claimed` flag hides manual claim button - no duplicate claims possible
  • Race condition: If both users open within milliseconds, API uses optimistic locking (first write wins)
  • Local state: After auto-claim API returns, update local Hive immediately so Steps screen shows "Claimed"

Overlay Display Logic

  • On app launch: Check if yesterday's steps qualify (combined >= 10,000) AND not yet shown to this user today
  • First user: `claimed == false` → call claim API → show "auto-claimed" overlay
  • Second user: `claimed == true` → show "partner claimed" overlay (no API call needed)
  • Track overlay shown: Store `lastOverlayShownDate` locally to prevent showing same celebration twice
  • Edge case: If both users trigger simultaneously, API ensures only one claim succeeds; both see appropriate overlay

Database Fields to Add

  • steps_days.claimed - boolean, whether LP was claimed
  • steps_days.claimed_by_user_id - UUID, who triggered the claim
  • steps_days.claimed_at - timestamp, when claimed

Local Hive: Add overlayShownForDate to track which day's celebration was displayed to this device.
