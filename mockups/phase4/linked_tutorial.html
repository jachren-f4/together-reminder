Phase 4.1: Linked Game Tutorial

Issue Identified

  • No tutorial for users unfamiliar with the Linked puzzle game
  • Users don't understand the drag-and-drop letter placement
  • Turn-based mechanic with partner isn't explained

Tutorial Structure (3 Steps)

  • Step 1: Read the Clues - Gray cells contain clues (emojis, text, or both). Arrows indicate answer direction.
  • Step 2: Drag Your Letters - Letter rack at bottom. DRAG letters onto empty grid cells. Can place as many or few as desired.
  • Step 3: Take Turns - Partners alternate turns. Tap Submit when ready and wait for partner.

Key Mechanics to Communicate

  • Drag, not tap: Letters are dragged from rack to grid (not tap-to-select)
  • Flexible placement: Can place as many or as few letters as desired before submitting
  • Turn-based: Wait for partner after submitting your turn
  • Shared puzzle: Both partners fill in the same grid together

Visual Elements

  • Spotlight effect: Highlights relevant area with pulsing border
  • Pointer finger: Animated hand pointing to spotlight
  • Progress dots: Show tutorial progress (1-3)
  • Skip button: Available on steps 1-2 for experienced users

When to Show

  • Show on first Linked game ever (check Hive storage)
  • Flag: hasSeenLinkedTutorial in user preferences
  • Optional: Add "?" help button in game header for replay

Implementation Notes

  • Create: app/lib/widgets/linked/linked_tutorial_overlay.dart
  • Add: _showTutorial state in linked_game_screen.dart
  • Store: StorageService().setHasSeenLinkedTutorial(true)
  • Check: On screen mount, show if !hasSeenLinkedTutorial

Spotlight Positioning

  • Step 1: Position over a clue cell (gray cell with arrow)
  • Step 2: Position over letter rack at bottom of screen
  • Step 3: No spotlight, general info overlay
